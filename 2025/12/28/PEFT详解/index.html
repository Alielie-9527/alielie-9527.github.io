PEFT详解

2025/12/28 LLM学习 PEFT 大模型微调

PEFT篇

注释：PEFT并不是某种具体的微调方法，而是一个技术集合

1.什么是PEFT

PEFT（Parameter-Efficient Fine-Tuning)即参数高效微调，其核心思想：不需要重新训练整个大模型，而是只训练其中很小的一部分参数，同时冻结绝大部分原有的参数

包含如下方法：

  1. LoRA目前流行
  2. QLoRA，模型先压缩，再训练
  3. P-Tuning/Prompt Tuning:让模型学习最佳提示词
  4. Prefix Tuning（前缀微调）transformer的每一层前面加入一些可训练的向量。

优点：

  1. 所需训练参数小
  2. 可以保存不同的插件，部署时融合即可，那么避免相同大模型存储多个副本。

2.前缀微调：通过连续前缀进行调节

​ 将前缀（其实模拟前面有输入），加到注意力机制的K和V中，模型参数保持不变。

首先看一下注意力机制：

Attention⁡(Q,K,V)=softmax⁡(QKTdk)V \operatorname{Attention}(Q, K, V) = \operatorname{softmax}\left( \frac{Q K^T}{\sqrt{d_k}} \right) V Attention(Q,K,V)=softmax(dk​​QKT​)V

线性投影定义：

Q=hWQ,K=hWK,V=hWV Q = h W_Q, \quad K = h W_K, \quad V = h W_V Q=hWQ​,K=hWK​,V=hWV​

通过拼接来修改：

K′=[PK;K]=[PK;hWK] K^{\prime} = [P_K; K] = [P_K; h W_K] K′=[PK​;K]=[PK​;hWK​] V′=[PV;V]=[PV;hWV] V^{\prime} = [P_V; V] = [P_V; h W_V] V′=[PV​;V]=[PV​;hWV​]

修改后的注意力机制：

Attention⁡(Q,K′,V′)=softmax⁡(Q(K′)Tdk)V′ \operatorname{Attention}(Q, K^{\prime}, V^{\prime}) = \operatorname{softmax}\left( \frac{Q (K^{\prime})^T}{\sqrt{d_k}} \right) V^{\prime} Attention(Q,K′,V′)=softmax(dk​​Q(K′)T​)V′

3.提示词微调

（1）方法的公式化表达

  • 输入序列：X={x1,x2,…,xn}X = \{x_1, x_2, \dots, x_n\}X={x1​,x2​,…,xn​}
  • Embedding 矩阵：E∈R∣V∣×dE \in \mathbb{R}^{|V| \times d}E∈R∣V∣×d （词表大小 ×\times× 维度）

正常流程：

  1. 将 XXX 转化为向量矩阵：Xe=Embed⁡(X)X_e = \operatorname{Embed}(X)Xe​=Embed(X)
  2. 输入模型：Output=LLM⁡(Xe)Output = \operatorname{LLM}(X_e)Output=LLM(Xe​)

Prompt Tuning 流程：

  1. 定义 Soft Prompt：初始化一个可训练的矩阵 PPP，形状为 [L,d][L, d][L,d]。
    • LLL：提示词长度（例如 20 个 token）。
    • ddd：模型的隐藏层维度（例如 4096）。
  2. 拼接（Concatenate）：将 PPP 拼在 XeX_eXe​ 前面。Input=[P;Xe] Input = [P; X_e] Input=[P;Xe​] 现在的输入矩阵形状变成了 (L+n)×d(L + n) \times d(L+n)×d。
  3. 输入模型：Output=LLM([P;Xe])Output = \text{LLM}([P; X_e])Output=LLM([P;Xe​])

（2）初始化策略：

  1. 随机初始化，使用小的随机值初始化。简单，但需要仔细调整学习率
  2. 词汇初始化，用模型词汇中与任务相关的词汇的平均嵌入来初始化，比如摘要的表达：summarize，Abstract。

（3）方法评估：

该方法对小模型效果差，大模型表现可以但是赶不上LoRA。并存在如下问题：

  1. 收敛慢、不稳定：直接随机初始化一组向量（Soft Prompt）去训练，很容易陷入局部最优，或者训练很久都不动。
  2. 独立性问题：Prompt Tuning 把每个虚拟 Token 当作独立的参数去学。但实际上，自然语言的词与词之间是有关联的（比如“New”后面接“York”概率大）。把虚拟 Token 视为完全独立的参数，忽略了它们之间的序列依赖性。

4.P-Tuning

（1）p-Tuning-v1:

​ 在提示词微调上做优化，在提示词微调中训练出来的提示词token之间没有关联性，于是在v1版，引入了Prompt Encoder：不直接去优化提示词向量，而是去优化一个能产生这些向量的神经网络。

架构流程：

  1. 输入层构建索引序列，非常简单的[0, 1, 2, ..., L-1]

  2. Prompt Encoder(提示编码器)：

    两部分组成（这两部分的参数都需要训练）：

    • Embedding层：索引变成向量
    • LSTM或者MLP，直白来说作用就是让上一步向量产生关联。

  3. 拼接原始输入的Embedding，输入大模型。

  4. 反向传播

推理阶段：

​ 训练完成后，我们只需要运行一次Prompt Encoder，然后保存生成的向量在后续推理过程中拼接即可。
